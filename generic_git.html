Основи Git

1. Коротка інформація про системи контролю версій та Git

Система контролю версій (VCS - Version Control System) дозволяє відстежувати зміни у файлах та керувати ними. Git — це одна з найпопулярніших розподілених систем контролю версій, яка дозволяє кожному учаснику зберігати повну копію репозиторію.

2. Основні можливості та переваги використання Git

  • Розподілена архітектура
  • Паралельна розробка
  • Відкат до попередніх версій
  • Протоколювання змін
  • Підтримка командної роботи
  • Ефективне злиття змін
  • Легка інтеграція з CI/CD

3. 10 базових команд Git

  1. git init - Ініціалізація нового репозиторію Git у поточній директорії. 
    git init
  2. git clone <url> - Клонування існуючого віддаленого репозиторію на локальний комп'ютер. 
    git clone https://github.com/username/repository.git
  3. git status - Показує статус робочого каталогу та індексу. 
    git status
  4. git add <file> - Додає файл у staging для наступного коміту. 
    git add file.txt
  5. git commit -m "Коментар" - Створює коміт з описом. 
    git commit -m "Додавання нової функції"
  6. git log - Показує журнал комітів. 
    git log
  7. git branch - Перегляд або створення нових гілок. 
    git branch
  8. git checkout <branch> - Перемикання між гілками. 
    git checkout new-feature
  9. git merge <branch> - Об'єднання змін з іншої гілки. 
    git merge new-feature
  10. git push origin <branch> - Відправка локальних змін у віддалений репозиторій. 
    git push origin main
